## Deep-research contribution: etcd / API server scalability

Sources validated:
- https://kubernetes.io/docs/setup/best-practices/cluster-large/
- https://kubernetes.io/docs/concepts/architecture/
- https://kubernetes.io/docs/concepts/architecture/controller/
- https://kubernetes.io/docs/concepts/cluster-administration/flow-control/
- https://kubernetes.io/docs/reference/instrumentation/metrics/
- https://kubernetes.io/docs/tasks/administer-cluster/configure-upgrade-etcd/
- https://etcd.io/docs/v3.6/op-guide/hardware/
- https://etcd.io/docs/v3.6/op-guide/performance/

Observed:
- Kubernetes v1.36 large-cluster guidance states support-shape limits of 110 pods per node, 5,000 nodes, 150,000 total pods, and 300,000 total containers.
- Kubernetes architecture docs describe kube-apiserver as horizontally scalable and etcd as the consistent, highly available backing store for cluster data.
- Kubernetes large-cluster guidance notes that Event objects can be stored in a separate dedicated etcd instance as an advanced performance option.
- Kubernetes etcd operation docs recommend dedicated machines or isolated environments for etcd and explicitly require a backup plan.
- etcd hardware guidance emphasizes disk write latency and gives sequential IOPS guidance: around 50 sequential IOPS minimum and around 500 sequential IOPS for heavily loaded clusters.
- Kubernetes API Priority and Fairness docs describe finer request classification, queueing, fair dispatch, and applicability to watch requests.
- Kubernetes controller docs frame controllers as control loops watching cluster state and moving actual state toward desired state.

Interpretation for k8s-info:
- Added a control-plane scalability guide that connects object count, watch fan-out, APF, etcd latency, and controller/informer behavior.
- Kept all guidance generic and public-source-grounded; no internal cluster size, metric value, topology, hostname, IP, dashboard, ticket, or incident detail was recorded.
- Treated official large-cluster numbers as support-shape boundaries, not as a direct production sizing promise.

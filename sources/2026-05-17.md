## Sources validated

### GitHub Copilot cloud agent

Source:
- https://docs.github.com/en/copilot/how-tos/agents/copilot-coding-agent

Observed:
- Page title validated as `GitHub Copilot cloud agent - GitHub Docs`.
- The documentation surface includes agent sessions, PR / issue workflow, MCP and cloud agent risks / mitigations, firewall / network settings, secrets and variables, activity / logs, and management topics.

Interpretation for k8s-info:
- Treat Copilot coding agent as a PR / issue / review assistant.
- Keep repository, MCP, network, secrets / variables, and output scope explicit.
- Do not treat agent-generated output as approved production change.

### Anthropic Claude Code security

Source:
- https://docs.anthropic.com/en/docs/claude-code/security

Observed:
- Page title validated as `Security - Claude Code Docs`.
- The security page includes default permissions, explicit permission for file/command actions, sensitive-code guidance, project-specific permission settings, dev containers, and credential-scope / blast-radius considerations.

Interpretation for k8s-info:
- Use least privilege and explicit approval as the default posture for SRE agent work.
- Prefer dev containers / throwaway workspaces for command execution.
- Never hand production credentials to an unbounded agent session.

### OpenAI Codex

Source:
- https://developers.openai.com/codex/

Observed:
- Page title validated as `Codex | OpenAI Developers`.
- The page describes Codex as helping debug, trace failures, suggest targeted fixes, automate refactoring / testing / migrations / setup, and the docs surface includes permissions, sandboxing, shell / file tools, MCP / connectors, and background-mode topics.

Interpretation for k8s-info:
- Good fit: debug reasoning, patch proposal, test automation, runbook drafting.
- Unsafe default: direct production `kubectl apply`, `helm upgrade`, node drain, or cloud API mutation without independent gates.

### Kubernetes debugging and secrets docs

Sources:
- https://kubernetes.io/docs/tasks/debug/debug-cluster/
- https://kubernetes.io/docs/tasks/debug/debug-application/debug-running-pod/
- https://kubernetes.io/docs/concepts/security/secrets-good-practices/

Observed:
- Kubernetes troubleshooting/debugging pages validate the normal operator signal set: object status, Events, `kubectl describe`, logs, and cluster / Pod checks.
- Kubernetes Secrets good-practices documentation provides the safety boundary for secret access and RBAC-oriented handling.

Interpretation for k8s-info:
- AI-assisted incident triage should consume sanitized observation packets, not raw kubeconfig, Secret manifests, service-account tokens, private dashboard links, or customer identifiers.
- Incident summaries must separate observed facts, hypotheses, missing evidence, and next checks.

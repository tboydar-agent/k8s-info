## Verified public signals

### Kubernetes upstream

```text
Source: Kubernetes official releases / patch releases / GitHub releases / version skew policy
Signal:
  - Latest GitHub release observed: v1.36.0.
  - Official releases page states Kubernetes maintains release branches for the most recent three minor releases.
  - Current maintained upstream minors observed: 1.36, 1.35, 1.34.
  - Patch release page lists:
      1.36 EOL: 2027-06-28
      1.35 latest patch observed: 1.35.3; next patch release: 1.35.4
      1.34 latest patch observed: 1.34.6
  - Version skew policy states:
      kubelet / kube-proxy must not be newer than kube-apiserver and may be up to three minor versions older.
      kube-controller-manager / kube-scheduler / cloud-controller-manager should match kube-apiserver minor, with one-minor older allowed during live upgrades.
      kubectl is supported within one minor version older or newer than kube-apiserver.
Impact:
  - 1.33 should now be treated as outside the latest-three upstream maintenance baseline.
  - Large platforms should refresh upgrade runway and freeze-window assumptions.
  - HA API server skew narrows the allowed versions for node components and clients during rollout.
Confidence: high
```

## Platform engineering interpretation

```text
For large-scale / on-prem / 7x24 platforms:

  upstream EOL
      is necessary baseline
      but not enough

  provider/vendor support
      changes what is supported
      but does not remove addon risk

  real upgrade readiness requires:
      - control plane + node skew review
      - CNI / CSI / CRI compatibility matrix
      - observability diff for metric rename/deprecation
      - SLO-gated rollout waves
      - rollback / freeze-window plan
      - vendor support channel check
```

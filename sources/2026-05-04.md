## 6) Deep-research contribution: CSI / Kubernetes storage failure modes

```text
Topic:
  CSI / Kubernetes storage failure modes

Why material:
  Storage incidents often get misread as a single PVC or Pod problem. Official docs show that the real path crosses PVC/PV binding, StorageClass policy, CSI controller sidecars, CSI node plugins, kubelet plugin registration, backend attach/mount, snapshot CRDs, and reclaim/finalizer behavior.

Contribution:
  Added docs/csi-storage-failure-modes.md as a source-grounded Traditional Chinese on-call 導讀.
```

Validated sources:

- https://kubernetes.io/docs/concepts/storage/persistent-volumes/
- https://kubernetes.io/docs/concepts/storage/storage-classes/
- https://kubernetes.io/docs/concepts/storage/volumes/
- https://kubernetes.io/docs/concepts/storage/volume-snapshots/
- https://kubernetes-csi.github.io/docs/
- https://kubernetes-csi.github.io/docs/sidecar-containers.html
- https://kubernetes-csi.github.io/docs/external-provisioner.html
- https://kubernetes-csi.github.io/docs/external-attacher.html
- https://kubernetes-csi.github.io/docs/external-resizer.html
- https://kubernetes-csi.github.io/docs/node-driver-registrar.html

Observed facts converted into operator workflow:

```text
- PV/PVC binding is exclusive and PVCs can remain unbound until a matching PV exists.
- StorageClass controls provisioner, reclaimPolicy, allowVolumeExpansion, volumeBindingMode, and topology constraints.
- CSI external-provisioner watches PVCs and calls CreateVolume / DeleteVolume.
- CSI external-attacher watches VolumeAttachment and calls ControllerPublishVolume / ControllerUnpublishVolume.
- kubelet talks directly to node CSI drivers for NodeStageVolume / NodePublishVolume through the plugin registration mechanism.
- VolumeSnapshot support uses CRDs, a snapshot controller, csi-snapshotter, and driver capability; it is not universal for every CSI driver.
```
